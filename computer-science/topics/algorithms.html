Searching Algorithms

```

Linear Search

Check each element one by one from the start until the target is found or the list is exhausted. Works on unsorted data.

``` def linear_search(lst, target): for i in range(len(lst)): if lst[i] == target: return i # Return index if found return -1 # Return -1 if not found
CaseComparisonsBig O
Best1 (target is first element)O(1)
Averagen/2O(n)
Worstn (not found or last element)O(n)
```

Binary Search

Repeatedly halve the search space. Requires a sorted list. Far more efficient than linear search for large datasets.

``` def binary_search(lst, target): low, high = 0, len(lst) - 1 while low <= high: mid = (low + high) // 2 if lst[mid] == target: return mid elif lst[mid] < target: low = mid + 1 # Target in right half else: high = mid - 1 # Target in left half return -1
```

Sorting Algorithms

```

Bubble Sort

Repeatedly compare adjacent pairs and swap if in the wrong order. Each pass "bubbles" the largest unsorted element to the end.

``` def bubble_sort(lst): n = len(lst) for i in range(n - 1): for j in range(n - 1 - i): if lst[j] > lst[j+1]: lst[j], lst[j+1] = lst[j+1], lst[j] return lst
```

Merge Sort

A divide-and-conquer algorithm. Recursively split the list in half, sort each half, then merge the sorted halves back together.

``` # Merge sort — O(n log n) def merge_sort(lst): if len(lst) <= 1: return lst mid = len(lst) // 2 left = merge_sort(lst[:mid]) right = merge_sort(lst[mid:]) return merge(left, right) def merge(left, right): result = [] i = j = 0 while i < len(left) and j < len(right): if left[i] <= right[j]: result.append(left[i]); i += 1 else: result.append(right[j]); j += 1 return result + left[i:] + right[j:]
```

Quick Sort

Choose a pivot, partition the list so all elements less than the pivot are to its left and all greater are to its right, then recursively sort each partition.

``` def quick_sort(lst): if len(lst) <= 1: return lst pivot = lst[len(lst) // 2] left = [x for x in lst if x < pivot] middle = [x for x in lst if x == pivot] right = [x for x in lst if x > pivot] return quick_sort(left) + middle + quick_sort(right)
```

Graph Algorithms

```

Depth-First Search (DFS)

Explore as far as possible along each branch before backtracking. Uses a stack (or recursion).

``` def dfs(graph, start, visited=None): if visited is None: visited = set() visited.add(start) print(start) for neighbour in graph[start]: if neighbour not in visited: dfs(graph, neighbour, visited)
```

Breadth-First Search (BFS)

Explore all neighbours at the current depth before moving deeper. Uses a queue. Finds the shortest path in an unweighted graph.

``` from collections import deque def bfs(graph, start): visited = {start} queue = deque([start]) while queue: node = queue.popleft() print(node) for neighbour in graph[node]: if neighbour not in visited: visited.add(neighbour) queue.append(neighbour)
```

Dijkstra's Algorithm

Finds the shortest path between a source node and all other nodes in a weighted graph (non-negative weights only).

Dijkstra's Steps 1. Set distance to source = 0, all others = infinity.
2. Visit the unvisited node with the smallest distance.
3. Update distances to its neighbours: if current distance + edge weight < recorded distance, update it.
4. Mark the node as visited. Repeat until all nodes are visited.

Time Complexity & Big O

Big O notation describes how an algorithm's runtime grows as the input size n increases. It expresses the worst case unless stated otherwise.

```
NotationNameExamplen=1000 (approx ops)
O(1)ConstantArray access, hash lookup1
O(log n)LogarithmicBinary search, BST search10
O(n)LinearLinear search, single loop1,000
O(n log n)LinearithmicMerge sort, quick sort (avg)10,000
O(n²)QuadraticBubble sort, nested loops1,000,000
O(2ⁿ)ExponentialBrute-force subset problems10^301
Reading Big O Drop constants and lower-order terms: O(3n² + 2n + 1) → O(n²). We only care about the dominant term as n grows large. Two nested loops over n elements = O(n²). Three nested = O(n³).
```

Recursion

A recursive function is one that calls itself. Every recursive solution must have a base case (stopping condition) and a recursive case that moves toward it.

```
Without a base case A recursive function without a base case will call itself forever, eventually causing a stack overflow when the call stack runs out of memory.
Classic Example — Factorial
Write a recursive function to calculate n! (n factorial).
1
Base case: 0! = 1 (and 1! = 1)
2
Recursive case: n! = n × (n-1)!
``` def factorial(n): if n == 0: # Base case return 1 return n * factorial(n - 1) # Recursive case # Call stack for factorial(4): # factorial(4) → 4 × factorial(3) # factorial(3) → 3 × factorial(2) # factorial(2) → 2 × factorial(1) # factorial(1) → 1 × factorial(0) # factorial(0) → 1 ← base case # Unwinds: 1→1→2→6→24
```

Worked Example — Fibonacci
Write a recursive function to return the nth Fibonacci number. Identify the inefficiency and suggest an improvement.
``` def fib(n): if n <= 1: # Base cases: fib(0)=0, fib(1)=1 return n return fib(n-1) + fib(n-2) # Inefficiency: fib(n-2) is recalculated many times # fib(5) calls fib(3) twice, fib(2) three times, etc. # Time complexity: O(2^n) — exponential! # Improvement: memoisation (cache results) from functools import lru_cache @lru_cache(maxsize=None) def fib_fast(n): if n <= 1: return n return fib_fast(n-1) + fib_fast(n-2) # Now O(n) — each value computed only once
