Network Types & Topologies

```

Types of Network

TypeScaleDescriptionExample
PAN~1mPersonal Area Network — devices around one personBluetooth headphones, smartwatch
LANBuildingLocal Area Network — single site, privately ownedSchool network, home Wi-Fi
WANGlobalWide Area Network — multiple sites, often leased infrastructureThe Internet, corporate VPN

Network Topologies

TopologyStructureAdvantagesDisadvantages
BusAll devices on one cable; terminators at each endCheap; easy to add devicesCollision-prone; one break kills the network; not scalable
StarAll devices connect to a central switch/hubFast; one cable failure only affects one device; easy to manageCentral switch is a single point of failure; more cable needed
RingEach device connects to next in a loopNo collisions (token-based); equal accessOne failure breaks the ring; adding devices is disruptive
MeshEvery device connects to many/all othersHighly resilient; multiple pathsVery expensive; complex to manage
Star is dominant in practice Almost all modern LANs use star topology with Ethernet switches. The switch learns which MAC addresses are on which ports and sends frames only to the correct destination — reducing collisions and improving efficiency vs a hub.

Wired vs Wireless

Wired (Ethernet)Wireless (Wi-Fi)
SpeedUp to 100GbpsUp to ~10Gbps (Wi-Fi 6)
SecurityMore secure — physical access neededLess secure — signals travel through air
ReliabilityVery reliableSubject to interference
FlexibilityRequires cable runsDevices can move freely
```

Protocols & the OSI Model

A protocol is an agreed set of rules for communication between devices. The OSI model (Open Systems Interconnection) divides networking into 7 layers, each with a specific role.

```
7
Application
User-facing protocols: HTTP, HTTPS, FTP, SMTP, DNS
6
Presentation
Data formatting, encryption, compression (SSL/TLS)
5
Session
Manages connections between applications; authentication
4
Transport
End-to-end delivery, error checking, flow control: TCP, UDP
3
Network
Logical addressing and routing: IP, ICMP, routers
2
Data Link
Physical addressing (MAC), framing, error detection: Ethernet, switches
1
Physical
Bits over medium: cables, Wi-Fi signals, voltage levels
Mnemonic for OSI layers (7→1) "All People Seem To Need Data Processing" — Application, Presentation, Session, Transport, Network, Data Link, Physical.

Why Layer?

Layering means each layer only needs to communicate with the layers directly above and below it. This allows different vendors to implement different layers independently — a Wi-Fi card (Physical/Data Link) works with any TCP/IP stack (Network/Transport), which works with any browser (Application).

TCP vs UDP

TCP (Transmission Control Protocol)UDP (User Datagram Protocol)
ConnectionConnection-oriented (3-way handshake)Connectionless
ReliabilityGuaranteed delivery; retransmits lost packetsNo guarantee; packets may be lost
OrderPackets reassembled in correct orderNo ordering guarantee
SpeedSlower due to overheadFaster — minimal overhead
Use casesWeb browsing, email, file transferVideo streaming, gaming, DNS, VoIP
```

TCP/IP & the Internet

```

IP Addresses

Every device on a network has an IP address — a unique identifier for routing packets to the right destination.

IPv4IPv6
Format4 octets: 192.168.1.18 groups of 4 hex digits: 2001:0db8:85a3::8a2e:0370:7334
Address space~4.3 billion (2³²)~340 undecillion (2¹²⁸)
StatusExhausted — NAT requiredGradually replacing IPv4

Subnetting & CIDR

A subnet mask divides an IP address into a network portion and a host portion. Written as /prefix (CIDR notation) or dotted decimal.

``` IP address: 192.168.1.45 Subnet mask: 255.255.255.0 (/24) Network: 192.168.1.0 (first 24 bits) Host range: 192.168.1.1 – 192.168.1.254 Broadcast: 192.168.1.255 Usable hosts: 254 (2^8 - 2)
```

Key Internet Protocols

ProtocolLayerPurpose
DNSApplicationResolves domain names to IP addresses (e.g. google.com → 142.250.x.x)
DHCPApplicationAutomatically assigns IP addresses, subnet mask, gateway to new devices
HTTP/HTTPSApplicationTransfer of web pages; HTTPS adds TLS encryption
SMTPApplicationSending email
IMAP/POP3ApplicationReceiving email (IMAP keeps on server; POP3 downloads)
FTPApplicationFile transfer between hosts
SSHApplicationSecure remote terminal access
Worked Example — DNS Resolution
Describe what happens when you type "www.example.com" into a browser and press Enter, up to the point where a TCP connection is established.
1
Browser checks its local DNS cache — if cached and not expired, use that IP and skip to step 5.
2
Query sent to the configured DNS resolver (usually your router or ISP's DNS server).
3
Resolver performs recursive lookup: root nameserver → .com TLD nameserver → example.com authoritative nameserver.
4
IP address returned and cached by the resolver (respecting TTL).
5
Browser initiates TCP 3-way handshake with the web server at that IP on port 443 (HTTPS): SYN → SYN-ACK → ACK.
6
TLS handshake establishes encrypted session. HTTP request is sent.
```

Network Security

```

Common Threats

ThreatDescription
MalwareMalicious software: viruses (self-replicating), worms (spread without host), trojans (disguised), ransomware (encrypts files), spyware
PhishingFraudulent emails/websites trick users into revealing credentials or downloading malware
DoS/DDoSFlood a server with requests to make it unavailable; DDoS uses many compromised machines (botnet)
Man-in-the-MiddleAttacker intercepts and potentially alters communications between two parties
SQL InjectionMalicious SQL inserted into input fields to manipulate databases
Brute ForceSystematically trying all possible passwords until correct one is found

Countermeasures

CountermeasureWhat it does
FirewallFilters incoming/outgoing traffic based on rules; blocks unauthorised access
Encryption (TLS/SSL)Scrambles data in transit so interceptors cannot read it
Hashing & salting passwordsStores a one-way hash so plaintext password is never stored; salt prevents rainbow table attacks
Two-factor authenticationRequires second proof of identity beyond password
Access control listsRestrict which users/devices can access which resources
Intrusion Detection System (IDS)Monitors network for suspicious patterns; alerts administrators
VPNEncrypts all traffic between a device and a private network over a public connection
SQL Injection Example Input: ' OR '1'='1 in a login form could produce: SELECT * FROM users WHERE username='' OR '1'='1' — which always returns true, bypassing authentication. Prevention: use parameterised queries / prepared statements.

Symmetric vs Asymmetric Encryption

SymmetricAsymmetric (Public Key)
KeysOne shared secret keyPublic key (encrypt) + private key (decrypt)
SpeedFastSlow
Key distributionProblem — how to share the key securely?Public key can be shared openly
UseBulk data encryption (AES)Key exchange, digital signatures (RSA)
In TLSUsed for session data once keys are exchangedUsed for initial key exchange

The Web & HTTP

```

Client-Server Model

The web uses a client-server model. Clients (browsers) make requests; servers respond with resources. Communication uses HTTP (or HTTPS for encrypted connections).

``` HTTP Request: GET /index.html HTTP/1.1 Host: www.example.com Accept: text/html HTTP Response: HTTP/1.1 200 OK Content-Type: text/html Content-Length: 1234 …
```

HTTP Status Codes

CodeMeaning
200OK — request successful
301/302Redirect — resource has moved
403Forbidden — no permission
404Not Found — resource doesn't exist
500Internal Server Error — server-side problem

How a Web Page Loads

When a browser receives an HTML response, it parses it and makes additional requests for each linked resource — CSS files, JavaScript, images. This is why a single page load can involve dozens of HTTP requests.

Cookies & Sessions HTTP is stateless — each request is independent. Cookies are small files stored on the client that the browser sends with every request to the same server, allowing the server to identify returning users and maintain session state (e.g. staying logged in, shopping basket contents).
```
