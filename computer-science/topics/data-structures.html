Arrays & Lists

An array is a static data structure that stores a fixed number of elements of the same data type in contiguous memory locations. Elements are accessed by index.

```
1D Array: scores[6]
Index:
0
1
2
3
4
5
Value:

85
91

94
scores[2] = 91  (highlighted)

2D Arrays

A 2D array stores data in rows and columns — like a grid or table. Accessed with two indices: grid[row][col].

``` # Declare and access a 2D array grid = [[1,2,3],[4,5,6],[7,8,9]] print(grid[1][2]) # Row 1, Col 2 → 6 # Traverse with nested loops for row in grid: for item in row: print(item, end=” “)
```
Static vs Dynamic Arrays are static — size is fixed at declaration. Python lists are dynamic — they can grow and shrink. AQA may ask you to distinguish between them and explain the memory implications.
```

Stacks

A stack is a Last In First Out (LIFO) data structure. Think of a stack of plates — you can only add or remove from the top.

```
Stack (top = index 3)
← TOP: "D"
"C"
"B"
"A"
OperationDescriptionAQA Pseudocode
PushAdd an item to the toppush(stack, item)
PopRemove and return the top itempop(stack)
Peek/TopView the top item without removing itpeek(stack)
isEmptyCheck if the stack is emptyisEmpty(stack)
isFullCheck if the stack is full (static)isFull(stack)
``` # Stack implementation using a list stack = [] stack.append(“A”) # Push stack.append(“B”) stack.append(“C”) top = stack.pop() # Pop → “C” peek = stack[-1] # Peek → “B”
```
Real-world uses of stacks Undo/redo in text editors, browser back button history, function call stack (tracking return addresses), expression evaluation, depth-first search.
Stack Overflow / Underflow Pushing to a full stack causes overflow. Popping from an empty stack causes underflow. Both must be checked in exam implementations.
```

Queues

A queue is a First In First Out (FIFO) data structure. Like a queue at a shop — items join at the rear and leave from the front.

```

10

40

OperationDescription
EnqueueAdd an item to the rear
DequeueRemove an item from the front
isEmpty / isFullCheck queue status

Circular Queue

A circular queue treats the array as a ring — when the rear pointer reaches the end it wraps back to index 0. This avoids wasting space left by dequeued items.

``` # Circular queue: rear wraps around rear = (rear + 1) % MAX_SIZE # Modulo keeps within bounds
```
Real-world uses of queues Print spoolers, CPU process scheduling, keyboard input buffer, breadth-first search, network packet routing.
```

Linked Lists

A linked list is a dynamic data structure where each element (node) stores a data value and a pointer to the next node. Nodes do not need to be in contiguous memory.

Worked Example — Inserting into a Linked List
Describe the steps to insert value 9 between nodes containing 7 and 21 in the list above.
1
Create a new node containing value 9.
2
Set new node's pointer to point to node 21 (the node after the insertion point).
3
Update node 7's pointer to point to the new node instead of node 21.
Result: Head → 5 → 12 → 7 → 9 → 21 → NULL

Trees & Graphs

```

Trees

A tree is a hierarchical data structure with a root node, branches, and leaves. Each node has at most one parent (except the root, which has none).

TermDefinition
RootThe top node — has no parent
NodeAn element in the tree containing data and pointers to children
LeafA node with no children
EdgeA connection between two nodes
SubtreeA node and all its descendants
HeightNumber of edges on the longest path from root to a leaf

Binary Search Tree (BST)

A BST is an ordered binary tree where for every node: all values in the left subtree are less, and all values in the right subtree are greater.

8 Insert order: 8, 3, 10, 1, 6, 14, 4, 7 / \ 3 10 Left subtree of 8: all < 8 / \ \ Right subtree of 8: all > 8 1 6 14 / \ 4 7
BST Search To find a value: start at root, go left if smaller, right if larger. Average O(log n) — much faster than linear search for large datasets. Worst case O(n) if the tree is unbalanced.

Graphs

A graph consists of vertices (nodes) connected by edges. Unlike trees, graphs can have cycles and multiple connections between nodes.

TypeDescriptionExample
UndirectedEdges have no direction — connection goes both waysFriendship network
Directed (digraph)Edges have a direction (arrows)Web page links, one-way roads
WeightedEdges have a numerical value (weight/cost)Road distances, network latency

Graph Representations

Adjacency listEach node stores a list of its neighboursSparse graphs, memory efficient

Hash Tables

A hash table stores key-value pairs. A hash function converts a key into an index (the hash), determining where the value is stored. This gives O(1) average lookup time.

```
``` Hash function example: index = key MOD table_size Key = 73, table size = 10: 73 MOD 10 = 3 → store at index 3 Key = 45, table size = 10: 45 MOD 10 = 5 → store at index 5 Key = 83, table size = 10: 83 MOD 10 = 3 → COLLISION with 73!
```

Collision Handling

MethodDescription
Linear probingIf slot is taken, try the next slot (index+1), then index+2, etc.
ChainingEach slot holds a linked list — colliding entries are chained together
Worked Example
Insert the following keys into a hash table of size 7 using the hash function key MOD 7 and linear probing for collisions: 14, 17, 31, 5, 21
``` 14 MOD 7 = 0 → place at index 0 17 MOD 7 = 3 → place at index 3 31 MOD 7 = 3 → COLLISION → try index 4 → place at 4 5 MOD 7 = 5 → place at index 5 21 MOD 7 = 0 → COLLISION → try 1 → place at index 1 Index: 0 1 2 3 4 5 6 Value: [14] [21] [ ] [17] [31] [5] [ ]
```
Why hash tables are powerful Average O(1) insertion, deletion and lookup — regardless of table size. Used in database indexing, caches, symbol tables in compilers, and implementing sets and dictionaries in Python.
