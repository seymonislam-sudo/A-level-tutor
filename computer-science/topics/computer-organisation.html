Von Neumann Architecture

The Von Neumann architecture (proposed by John von Neumann in 1945) is the foundation of virtually every modern computer. Its key principle is the stored program concept — both instructions and data are stored together in the same memory, encoded as binary.

```
Von Neumann Architecture
Control Unit (CU)
⟷
ALU
⟷
Registers
↑↓ via system buses
Main Memory (RAM)
⟷
I/O Devices

Key Components

ComponentFunction
Control Unit (CU)Directs operations of the CPU — fetches, decodes and executes instructions; coordinates all other components
Arithmetic Logic Unit (ALU)Performs arithmetic operations (+, -, ×, ÷) and logical operations (AND, OR, NOT, XOR, comparisons)
RegistersSmall, ultra-fast storage locations inside the CPU for immediate use during processing
Main Memory (RAM)Stores currently running programs and their data
BusesElectrical pathways connecting components — carry data, addresses and control signals

CPU Registers

RegisterFull NamePurpose
PCProgram CounterHolds the memory address of the next instruction to fetch
MARMemory Address RegisterHolds the address in memory currently being read from or written to
MDRMemory Data RegisterHolds the data that has just been fetched from or is about to be written to memory
CIRCurrent Instruction RegisterHolds the instruction currently being decoded and executed
ACCAccumulatorHolds the result of the most recent ALU operation
SPStack PointerPoints to the top of the call stack in memory
The Stored Program Concept Before Von Neumann, computers were physically rewired to change programs. The stored program concept means a computer can run any program loaded into memory — it doesn't need to be rebuilt. This is why the same CPU can run a game, a spreadsheet, and a compiler.
```

The Fetch-Execute Cycle

The CPU continuously repeats the fetch-execute cycle (also called the fetch-decode-execute cycle) to process instructions.

```
``` FETCH: MAR ← PC (copy address of next instruction to MAR) MDR ← Memory[MAR] (fetch instruction from memory into MDR) CIR ← MDR (copy instruction to CIR) PC ← PC + 1 (increment PC to point to next instruction) DECODE: CU decodes the instruction in CIR Identifies opcode (what to do) and operand (what to do it to) EXECUTE: CU sends control signals to relevant components ALU performs any arithmetic/logic Result stored in ACC or written back to memory
```
Worked Example — Tracing the Cycle
Trace the fetch-execute cycle for the instruction at address 200: LOAD 500 (load the value stored at address 500 into the accumulator). Assume PC = 200.
1
Fetch: MAR ← 200; MDR ← contents of address 200 (= "LOAD 500"); CIR ← "LOAD 500"; PC ← 201
2
Decode: CU identifies opcode = LOAD, operand = 500
3
Execute: MAR ← 500; MDR ← contents of address 500; ACC ← MDR
Common Exam Mistake Students often forget that after fetching, PC is incremented before execute. Also remember: the MAR is used twice in a LOAD instruction — once to fetch the instruction itself, and again to fetch the data from the operand address.
```

CPU Performance Factors

```
FactorDescriptionEffect of increasing
Clock speedNumber of fetch-execute cycles per second (Hz)More instructions executed per second; more heat generated
Number of coresIndependent processing units on the same chipTrue parallel execution of multiple threads; scales with parallelisable tasks
Cache sizeFast on-chip memory storing recently used data/instructionsFewer slow RAM accesses; reduced bottleneck
Word sizeAmount of data the CPU processes in one operation (32-bit vs 64-bit)Larger word = more data per instruction = faster for large datasets

Cache Memory Hierarchy

Cache is organised in levels — L1 is fastest and smallest, L3 is slowest and largest:

``` L1 cache: ~32KB per core — fastest (~1ns access) — checked first L2 cache: ~256KB per core — fast (~5ns access) L3 cache: ~8MB shared — slower (~20ns access) RAM: GB range — slow (~100ns access) — checked last
```
Pipelining Modern CPUs use pipelining — while one instruction is being executed, the next is being decoded, and the one after is being fetched. This overlaps stages so the CPU can process multiple instructions simultaneously, dramatically increasing throughput.

Multicore vs Clock Speed

Simply increasing clock speed generates excessive heat and power consumption — this is why CPU manufacturers moved to multicore designs. A dual-core CPU at 2GHz can outperform a single-core at 3GHz for parallelisable workloads, while using less power.

```

Instruction Sets & Addressing Modes

An instruction set architecture (ISA) defines the set of instructions a CPU can execute, the registers available, and the data formats supported. It is the interface between hardware and software.

```

Instruction Format

A machine code instruction consists of two parts:

``` | OPCODE | OPERAND | | 4 bits | 12 bits | OPCODE — specifies the operation (e.g. LOAD, ADD, STORE, JUMP) OPERAND — specifies the data or address to operate on
```

Addressing Modes

ModeDescriptionExample
ImmediateThe operand IS the data value itselfADD #5 → add 5 to ACC
DirectThe operand is the memory address where data is storedLOAD 200 → load value at address 200
IndirectThe operand holds an address that points to another address containing the dataLOAD @200 → address 200 holds another address
IndexedThe operand is added to an index register to calculate the actual addressLOAD 200,IX → address = 200 + IX

Worked Example — Assembly Program
Write AQA assembly code to calculate the sum of two numbers stored at addresses 100 and 101, and store the result at address 102.
LDR R0, 100 ; Load value at address 100 into R0 LDR R1, 101 ; Load value at address 101 into R1 ADD R2, R0, R1 ; R2 = R0 + R1 STR R2, 102 ; Store result at address 102 HALT ; Stop
```

CISC vs RISC

There are two major philosophies for instruction set design:

```
CISC (Complex Instruction Set Computer)RISC (Reduced Instruction Set Computer)
InstructionsMany complex instructions; each can do a lotSmall set of simple instructions
Cycles per instructionVariable — some take many cyclesUsually 1 cycle per instruction
Code sizeFewer instructions needed for a taskMore instructions needed
Hardware complexityComplex decoder; harder to pipelineSimple decoder; easy to pipeline
RegistersFewer registersMany registers
Memory accessInstructions can access memory directlyOnly LOAD/STORE access memory (load-store architecture)
Examplesx86 (Intel/AMD desktop CPUs)ARM (phones, tablets, Apple M-series)
Why ARM dominates mobile RISC's simplicity means fewer transistors, lower power consumption, and less heat. This makes ARM-based chips ideal for battery-powered devices. Apple's M-series chips prove RISC can match CISC performance while using a fraction of the power.
```

```

The Three Buses

BusDirectionCarriesWidth effect
Address busUnidirectional (CPU → memory/devices)Memory addressesMore bits = larger addressable memory (32-bit = 4GB max)

Control busBidirectionalControl signals (read/write, clock, interrupts)More lines = more control signals possible

Interrupts

An interrupt is a signal sent to the CPU by hardware or software requesting immediate attention. After each fetch-execute cycle, the CPU checks an interrupt register.

``` If interrupt pending AND priority > current task: 1. Save current state to stack (registers, PC) 1. Load ISR address into PC (from interrupt vector table) 1. Execute ISR (Interrupt Service Routine) 1. Restore saved state from stack 1. Continue previous task
```
Interrupt typeExample
Hardware interruptKeyboard keypress, mouse click, disk read complete
Software interruptProgram requesting OS service (system call)
Exception/faultDivision by zero, illegal memory access, overflow
Why interrupts matter Without interrupts, the CPU would have to constantly poll every device to check if it needs attention — wasting millions of cycles. Interrupts allow the CPU to do useful work and only respond to devices when they actually need it.
```
