Hardware Components

Hardware refers to the physical components of a computer system. Every computing device — from a phone to a supercomputer — is built on the same fundamental components.

```

Internal Components

ComponentFunction
CPU (Central Processing Unit)Executes program instructions — the "brain" of the computer
RAM (Random Access Memory)Volatile temporary storage — holds running programs and data
ROM (Read-Only Memory)Non-volatile permanent storage — holds firmware/BIOS
CacheVery fast memory between CPU and RAM — stores frequently used data
MotherboardMain circuit board — connects all components via buses
GPUGraphics Processing Unit — handles visual output, parallel processing
PSUPower Supply Unit — converts mains power to voltages the components need

Storage Devices

TypeVolatile?SpeedCapacityUse
RAMYesVery fastGBRunning programs
CacheYesFastestKB–MBCPU working data
SSD (Solid State Drive)NoFastGB–TBOS, applications, files
HDD (Hard Disk Drive)NoSlowerTBMass storage
Optical (CD/DVD/Blu-ray)NoSlowGBDistribution, archiving
Flash/USBNoModerateGBPortable storage

Input & Output Devices

Input devicesOutput devices
Keyboard, mouse, touchscreen, microphone, webcam, barcode scanner, sensorMonitor, printer, speaker, actuator, projector
Volatile vs Non-volatile Volatile memory loses its contents when power is removed (RAM, cache). Non-volatile memory retains data without power (SSD, HDD, ROM, flash). This is why you must save work — moving it from volatile RAM to non-volatile storage.
```

Software & Operating Systems

Software is the set of programs and data that run on hardware. It is divided into two broad categories:

```
CategoryDefinitionExamples
System softwareManages hardware and provides a platform for application softwareOperating system, device drivers, utilities, compilers
Application softwarePerforms tasks for the end userWord processors, games, browsers, databases

Types of Operating System

TypeDescriptionExample use
Multi-taskingRuns multiple programs apparently simultaneously using time-slicingDesktop OS (Windows, macOS, Linux)
Multi-userSupports multiple users concurrentlyServer OS, mainframes
Real-time (RTOS)Guarantees response within a strict time limitAircraft control, pacemakers, industrial automation
DistributedManages resources across a network of computersCloud computing platforms
EmbeddedDedicated, minimal OS built into a deviceSmart TV, washing machine, router
Why RTOS matters In a real-time system, a late response can be as bad as no response. A pacemaker that reacts 100ms too slowly could be fatal. RTOS schedulers guarantee maximum response times — something general-purpose OSes cannot do.
```

OS Functions in Detail

The operating system acts as an intermediary between user applications and hardware. Its core responsibilities are:

```

Process Management

The OS manages processes — programs in execution. It uses scheduling algorithms to decide which process gets CPU time and for how long.

SchedulerDescriptionAdvantage
Round RobinEach process gets a fixed time slice in rotationFair — no process starves
First Come First ServedProcesses run in order of arrivalSimple
Shortest Job FirstShortest estimated process runs nextMinimises average wait time
Priority SchedulingHigher-priority processes run firstImportant tasks get fast response

Memory Management

The OS manages how RAM is allocated between processes. Key techniques include:

TechniqueDescription
PagingMemory divided into fixed-size pages; processes don't need contiguous memory
SegmentationMemory divided into variable-size segments based on logical program sections
Virtual memoryDisk space used as an extension of RAM — slower but allows larger programs to run
Thrashing If a system relies too heavily on virtual memory, it spends more time swapping pages in and out of disk than actually executing programs — this is called thrashing. The solution is to add more RAM or reduce the number of running processes.

File Management

The OS provides a file system — a structured way to store, name, organise and retrieve files. It manages directories, permissions, metadata and handles reading/writing to storage devices.

Device Management

Device drivers are software that allow the OS to communicate with specific hardware devices. The OS provides a standard interface so applications don't need to know hardware-specific details.

Security & User Management

The OS enforces access control — authentication (who you are) and authorisation (what you're allowed to do). It isolates processes so one program can't read another's memory.

```

Programming Languages

```
LevelExampleCharacteristics
Machine code01001010 11000011Binary; directly executed by CPU; hardware-specific; very fast
Assembly languageMOV AX, 5 / ADD AX, BXMnemonics for machine code; one-to-one with instructions; still hardware-specific
High-level languagePython, Java, C++Human-readable; portable; must be translated; abstracts hardware

Imperative vs Declarative

ParadigmDescriptionExamples
Imperative (procedural)Describes how to do something — step-by-step instructionsPython, C, Pascal
Object-orientedOrganises code around objects with state and behaviourJava, Python, C++
Declarative (functional)Describes what the result should be, not how to compute itHaskell, SQL, Prolog
Why use assembly language? Assembly is used for device drivers, embedded systems, and performance-critical code where the programmer needs direct hardware control. It is faster than high-level code but much harder to write and maintain.
```

Translators & Compilation

High-level code must be translated into machine code before it can run. There are three types of translator:

```
TranslatorHow it worksOutputExamples
CompilerTranslates entire source code to machine code in one goStandalone executable fileC, C++, Go
InterpreterTranslates and executes one line at a time — no output fileNone (executes directly)Python (default), Ruby
AssemblerConverts assembly language mnemonics to machine codeObject codeNASM, MASM

Compiler vs Interpreter

CompilerInterpreter
Speed of executionFaster — already translatedSlower — translates on the fly
Development speedSlower — must recompile after changesFaster — run immediately
Error reportingAll errors reported after full compilationStops at first error — easier to debug
Source code distributionNot needed — distribute executableSource code required
PortabilityCompiled for specific platformInterpreter handles portability

Stages of Compilation

A compiler works through several stages to convert source code into machine code:

``` 1. Lexical analysis — tokenise source code; remove whitespace/comments 1. Syntax analysis — check grammar using parse tree (context-free grammar) 1. Semantic analysis — check meaning (type checking, scope, declarations) 1. Code generation — produce intermediate or target machine code 1. Code optimisation — improve efficiency (remove redundant code, etc.)
Worked Example — Lexical Analysis
Show how the lexer would tokenise the statement: total = price * quantity
1
"total" → IDENTIFIER (token type: variable name)
2
"=" → ASSIGNMENT_OPERATOR
3
"price" → IDENTIFIER
4
"*" → ARITHMETIC_OPERATOR (multiply)
5
"quantity" → IDENTIFIER
JIT Compilation Just-In-Time (JIT) compilation is a hybrid approach — code is compiled at runtime, just before it is needed. Java's JVM and Python's PyPy use JIT. It combines the portability of interpretation with much of the speed of compilation.
