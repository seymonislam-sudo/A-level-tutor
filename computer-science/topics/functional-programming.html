Functional Programming

A programming paradigm that treats computation as the evaluation of mathematical functions. No loops, no mutable state — just functions transforming data. Increasingly important in modern software.

Programming Paradigms

A programming paradigm is a style or approach to programming. AQA requires you to understand and contrast three:

Imperative / Procedural

Describes how to do something step by step. Uses variables, loops, and assignment. State changes over time.

Example: Python with for loops, variables updated in place.

Object-Oriented (OOP)

Groups data and behaviour into objects. Uses classes, inheritance, encapsulation. State lives inside objects.

Example: Python classes, Java.

Functional

Describes what to compute, not how. Built from pure functions with no side effects. No mutable state — data is never changed, only transformed into new data.

Examples: Haskell (purely functional), Python/JavaScript support functional style.

Key Characteristics of Functional Programming
  • Functions are first-class citizens — treated like any other value
  • No side effects — a function only returns a value, never modifies external state
  • Immutability — data cannot be changed once created
  • Referential transparency — calling a function with the same inputs always gives the same output
  • No loops — iteration is replaced with recursion or higher-order functions

First-Class Functions

In functional programming, functions are first-class citizens. This means a function can be:

  • Assigned to a variable
  • Passed as an argument to another function
  • Returned as the result of a function
# Functions assigned to variables double = lambda x: x * 2 square = lambda x: x ** 2 print(double(5)) # 10 print(square(4)) # 16 # Passing a function as an argument def apply(func, value): return func(value) print(apply(double, 7)) # 14 print(apply(square, 3)) # 9

A lambda (or anonymous function) is a function defined without a name, typically used when a short function is needed in one place only.

# Lambda syntax: lambda parameters: expression add = lambda x, y: x + y print(add(3, 4)) # 7

Higher-Order Functions & Composition

A higher-order function is a function that either:

  • Takes one or more functions as arguments, or
  • Returns a function as its result

Function Composition

Combining two or more functions so the output of one becomes the input of the next.

def compose(f, g): return lambda x: f(g(x)) double = lambda x: x * 2 add_one = lambda x: x + 1 double_then_add = compose(add_one, double) print(double_then_add(5)) # (5 * 2) + 1 = 11

Partial Application

Creating a new function by fixing some arguments of an existing function. The new function takes fewer arguments.

from functools import partial def power(base, exponent): return base ** exponent square = partial(power, exponent=2) cube = partial(power, exponent=3) print(square(4)) # 16 print(cube(3)) # 27
Currying vs Partial Application Currying transforms a function of multiple arguments into a chain of single-argument functions: f(a, b) becomes f(a)(b).

Partial application fixes some (not all) arguments of a function, returning a new function.

AQA expects you to understand both concepts and be able to trace through examples.

Map, Filter & Reduce / Fold

These three higher-order functions are the workhorses of functional programming. They replace most loops.

Map

Applies a function to every element of a list, returning a new list of the same length.

numbers = [1, 2, 3, 4, 5] doubled = list(map(lambda x: x * 2, numbers)) print(doubled) # [2, 4, 6, 8, 10] # Equivalent without map (imperative style): # result = [] # for x in numbers: # result.append(x * 2)

Filter

Returns a new list containing only the elements for which the function returns True.

numbers = [1, 2, 3, 4, 5, 6] evens = list(filter(lambda x: x % 2 == 0, numbers)) print(evens) # [2, 4, 6]

Reduce / Fold

Combines all elements of a list into a single value by repeatedly applying a function to a running accumulator.

from functools import reduce numbers = [1, 2, 3, 4, 5] total = reduce(lambda acc, x: acc + x, numbers) print(total) # 15 (1+2+3+4+5) # How it works step by step: # acc=1, x=2 → acc=3 # acc=3, x=3 → acc=6 # acc=6, x=4 → acc=10 # acc=10, x=5 → acc=15
▶ Worked Example — Chaining Map and Filter

Given a list of numbers, produce a list of the squares of all odd numbers.

1
Use filter to keep only odd numbers.
2
Use map to square each remaining number.
3
nums = [1, 2, 3, 4, 5, 6] odds = filter(lambda x: x % 2 != 0, nums) odd_squares = list(map(lambda x: x ** 2, odds)) print(odd_squares) # [1, 9, 25]

Immutability & No Side Effects

Pure Functions

A pure function always produces the same output for the same input, and has no side effects — it does not modify any external state, print to the screen, write to a file, or change any variable outside itself.

Impure (side effects)

total = 0 def add_to_total(x): global total total += x # modifies external state! return total

Pure (no side effects)

def add(a, b): return a + b # only uses inputs # nothing modified outside

Immutability

In functional programming, once a variable is assigned a value, it cannot be changed. Instead of modifying data, you create new data.

# Imperative (mutable) style: numbers = [1, 2, 3] numbers.append(4) # original list is modified # Functional (immutable) style: numbers = [1, 2, 3] new_numbers = numbers + [4] # original unchanged, new list created
Why Immutability Matters
  • Easier to reason about — data never changes unexpectedly
  • Thread-safe — no risk of two processes modifying the same data simultaneously
  • Easier to test — pure functions with same inputs always give same outputs
  • No debugging side effects — bugs don't hide in global state
