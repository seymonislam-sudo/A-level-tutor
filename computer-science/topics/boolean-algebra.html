Logic Gates

A logic gate is a physical electronic component that implements a Boolean function. Each gate takes one or more binary inputs (0 or 1) and produces a single binary output.

AND
A · B  or  A AND B
Output is 1 only when both inputs are 1.
OR
A + B  or  A OR B
Output is 1 when at least one input is 1.
NOT
¬A  or  NOT A  or  A'
Inverts the input. 0 becomes 1, 1 becomes 0.
NAND
¬(A · B)
NOT AND — output is 0 only when both inputs are 1. A universal gate.
NOR
¬(A + B)
NOT OR — output is 1 only when both inputs are 0. Also a universal gate.
XOR
A ⊕ B
Exclusive OR — output is 1 when inputs are different.
Universal Gates NAND and NOR are called universal gates because any other logic gate (AND, OR, NOT, XOR) can be built using only NAND gates, or only NOR gates. This is important in chip manufacturing — you only need one type of gate.

Boolean Algebra Notation

Boolean algebra uses standard notation to write logical expressions compactly.

OperationSymbolExampleMeaning
AND·  or  no symbolA · B  or  ABA AND B
OR+A + BA OR B
NOT¬  or  overbar  or  '¬A  or  A'NOT A
XOR⊕A ⊕ BA XOR B
Operator Precedence Like BIDMAS in maths, Boolean algebra has precedence rules:

1. Brackets first
2. NOT (highest precedence among operators)
3. AND
4. OR (lowest precedence)

So A + B · C means A + (B · C), not (A + B) · C.

Boolean Laws & Identities

These laws let you manipulate and simplify Boolean expressions. You must know them — they are the tools used in simplification questions.

LawAND formOR form
IdentityA · 1 = AA + 0 = A
Null / DominationA · 0 = 0A + 1 = 1
IdempotentA · A = AA + A = A
ComplementA · ¬A = 0A + ¬A = 1
Double Negation¬(¬A) = A
CommutativeA · B = B · AA + B = B + A
Associative(A·B)·C = A·(B·C)(A+B)+C = A+(B+C)
DistributiveA·(B+C) = A·B + A·CA+(B·C) = (A+B)·(A+C)
AbsorptionA·(A+B) = AA + A·B = A
De Morgan's¬(A·B) = ¬A + ¬B¬(A+B) = ¬A · ¬B
De Morgan's Laws — Most Important De Morgan's laws are the most commonly examined. They convert between AND and OR forms:

"Break the bar, change the operator"

¬(A · B) = ¬A + ¬B    (NOT of AND = OR of NOTs)
¬(A + B) = ¬A · ¬B    (NOT of OR = AND of NOTs)

Simplification

Simplifying Boolean expressions reduces the number of gates needed in a circuit — saving cost, power, and space. Apply the laws step by step, stating which law you use at each stage.

▶ Worked Example 1 — Simplification

Simplify: A · B + A · ¬B

1
Factor out A:   A · (B + ¬B)   [Distributive law]
2
B + ¬B = 1   [Complement law]
3
A · 1 = A   [Identity law] ✓
▶ Worked Example 2 — De Morgan's

Simplify: ¬(¬A · ¬B)

1
Apply De Morgan's: ¬(¬A · ¬B) = ¬(¬A) + ¬(¬B)
2
Apply Double Negation: = A + B ✓
▶ Worked Example 3 — Multi-step

Simplify: A · B + A · B · C + A · ¬B

1
Apply Absorption to first two terms: A·B + A·B·C = A·B   [A + A·C = A]
2
Expression becomes: A·B + A·¬B
3
Factor out A: A·(B + ¬B) = A·1 = A ✓
